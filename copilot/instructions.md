# BrokerReport — инструкции для агента

## Архитектура проекта

- `run.py` — точка входа: импорт HTML-отчётов + запуск Flask на `127.0.0.1:5000`
- `app.py` — Flask-приложение: роуты, фоновый дозор файлов
- `parser.py` — парсинг HTML-отчётов брокера (BeautifulSoup + lxml)
- `db.py` — SQLite модель + аналитические запросы
- `templates/` — Bootstrap 5 шаблоны (dashboard.html, report.html, base.html)
- `lua/prices.lua` — Lua-скрипт для QUIK: отправляет текущие цены инструментов на `/api/price`
- `lua/trades.lua` — Lua-скрипт для Kafka/AMQP (из другого проекта, справочно)

## API endpoints

- `POST /api/price` — принять цену инструмента (JSON: `sec_code`, `price`, `qty`, `class_code`) или пачку (`{"prices": [...]}`)
- `GET /api/prices` — получить все текущие цены
- `GET /api/report/<id>/profit` — прибыль по сделкам
- `GET /api/report/<id>/open` — открытые позиции

## База данных SQLite

Таблицы: `report`, `trade`, `repo`, `cash_flow`, `portfolio`, `financial_result`, `current_price`

**ВАЖНО**: сделки могут дублироваться между отчётами (один и тот же trade_id в отчётах за пересекающиеся периоды). При агрегации по всем отчётам (`report_id=None`) нужно дедуплицировать сделки по `(security_code, deal_number)`.

## Расчёт прибыли по трейд-сделкам

Используется **LIFO** (последняя покупка → продажа), а не FIFO. Каждая продажа матчится с самой **последней по времени покупкой** (последний элемент очереди), а не с самой старой.

Код в `_match_trades_lifo()`:
- Все транзакции обрабатываются в хронологическом порядке (перемешанные покупки и продажи)
- При получении покупки → добавляется в конец очереди
- При получении продажи → списывается из **конца** очереди (`buy_queue[-1]`), а не из начала
- Начальные позиции (акции, бывшие в портфеле на начало периода) НЕ учитываются в прибыли

## Фильтр по датам

Даты в формате `DD.MM.YYYY`. Для SQL-фильтрации используется преобразование в `YYYYMMDD` через `substr()`.

## Фоновый дозор

`app._watch_folder()` — поток-demon, раз в 60 секунд проверяет новые `.html`/`.HTML` файлы.
Отслеживает уже импортированные файлы через `_imported_files = set()` на уровне модуля.

## Страницы

- `/` — дашборд со сводной аналитикой по всем отчётам + фильтр по датам
- `/report/<id>` — просмотр конкретного отчёта

## Типичные ошибки (избегать!)

1. **Не суммировать покупки и продажи лоб в лоб** — нужно LIFO-матчинг, а не просто total_buy - total_sell
2. **Не добавлять начальные позиции из портфеля** — они не являются трейд-операциями
3. **Не усреднять цены** — каждый лот считается индивидуально, а не средними
4. **Не забывать дедуплицировать сделки** при агрегации по всем отчётам
5. **Платежи по РЕПО** (проценты + комиссии) — добавлять отдельной строкой в итоговый результат
